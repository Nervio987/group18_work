# 智汇办公 AI 智能平台

> 基于 RAG + Agent + LangGraph 的企业级智能知识助手，聚焦"知识找人"的下一代办公体验。

---

## 一、项目背景

### 1.1 行业痛点

企业日常运营中，员工面对规章、流程、技术、行政等大量问题，传统解决方案存在以下核心问题：

| 痛点 | 描述 | 严重程度 |
|------|------|----------|
| **知识分散** | 制度散落在 OA、文档、邮件、IM 群中，新员工入职平均**2-3 周**才能独立工作 | 🔴 极痛 |
| **检索低效** | 找一份报销流程可能要在 5 个系统切换、问 3 个同事 | 🟠 高频 |
| **理解门槛** | 制度文件晦涩，HR/IT 每天被重复咨询**30+ 次**相同问题 | 🟠 高频 |
| **信息孤岛** | 跨团队（前端/后端/数据/运维）知识共享困难 | 🔴 极痛 |
| **更新滞后** | 文档更新到员工看到平均延迟**1-2 周**，错过重要变更 | 🟡 中等 |
| **个性化缺失** | 同一份文档无法针对员工角色推送对应内容 | 🟡 中等 |

### 1.2 现状分析

| 现有方案 | 问题 |
|----------|------|
| 企业 Wiki（Confluence 等） | 全文搜索，准确率低，需要用户自己"猜关键词" |
| FAQ 列表 | 静态陈列，新增/变更问题无法自动覆盖 |
| IM 群答疑 | 答案仅限群成员可见，无法沉淀复用 |
| 传统搜索引擎（ES） | 仅关键词匹配，无法回答"为什么 / 怎么做" |

### 1.3 AI 带来的变革

| 维度 | 传统方案 | AI 智能助手 |
|------|---------|-----------|
| 检索方式 | 关键词模糊匹配 | 语义理解 + 关键词混合检索 |
| 答案生成 | 列出文档片段 | 综合多个文档生成**自然语言回答** |
| 知识来源 | 单一系统 | 多源融合（本地知识库 + 联网搜索 + 文档/Excel/网页） |
| 交互方式 | 菜单/表单 | 自然语言对话 + 多轮上下文 |
| 准确度 | 依赖关键词 | 引用溯源 + Rerank 精排 |
| 工具能力 | 无 | 联网搜索、文档生成、数学计算、文件分析 |

## 二、产品定位

**智汇办公**是一个面向中小企业的 **AI 智能办公助理平台**，让员工通过自然语言对话即可获取企业知识、完成日常办公任务。

### 2.1 目标用户

| 角色 | 使用场景 | 价值 |
|------|---------|------|
| **新员工** | 入职快速上手 | 减少 60% 培训成本 |
| **HR/IT/行政** | 减少重复答疑 | 每天节省 2 小时 |
| **一线员工** | 日常流程查询 | 7×24h 即时响应 |
| **管理层** | 数据查询、报告生成 | 一句话生成 Word/PDF 报告 |

### 2.2 商业模式

- **B 端 SaaS 订阅**：按企业规模阶梯定价
- **私有化部署**：面向中大型企业，一次性买断 + 运维服务
- **增值服务**：行业知识库定制、微调模型训练、API 集成

## 三、核心功能

### 3.1 四大对话模块

平台提供 **4 个智能模块** + **1 个通用自由问答**，每个模块独立配置 RAG 知识库：

| 模块 | 图标 | 应用场景 | 知识库 |
|------|------|---------|--------|
| **规章制度** | 📋 | 报销、考勤、请假、绩效 | 制度文档 |
| **产品技术** | 💻 | API 文档、架构、运维 | 技术手册 |
| **行政服务** | 🏢 | 场地、IT 设备、福利 | 行政手册 |
| **自由问答** | 💬 | 任何开放性问题 | 通用知识库 + 用户个人库 |

### 3.2 三种 AI 推理模式

平台支持 **RAG、Agent、LangGraph** 三种推理模式，**任意模块可任意切换**：

| 模式 | 适用场景 | 能力 |
|------|---------|------|
| **RAG** | 知识库问答 | 检索增强生成 + 来源溯源 |
| **Agent** | 复杂任务 | 工具调用（联网/计算/文档生成/文件分析） |
| **LangGraph** | 多步推理 | 图编排 + 状态管理 + 长短期记忆 |

### 3.3 智能体工具集

Agent 模式内置 **6 类工具**：

1. **联网搜索**：超出本地知识库时自动启用
2. **数学计算**：复杂表达式自动求解
3. **文档生成**：一句话生成 Word / PDF 报告
4. **文件分析**：上传 PDF/Excel/Word 智能解读
5. **日期/系统查询**：实时信息获取
6. **幻觉检测**：检测 LLM 假装调用的兜底机制

### 3.4 个人知识库

普通用户可以**自助创建个人知识库**，自由问答模式可选用：

- 上传自己的文档（PDF、Word、Markdown、Excel、TXT）
- 仅自己可见，保护隐私
- RAG 检索时自动混入通用库

### 3.5 流式响应

所有 AI 输出均采用 **SSE 流式响应**，实时显示 AI 思考过程，**告别"长时间黑屏等待"**。

### 3.6 对话历史与收藏

- 自动持久化所有对话到后端
- 跨页面/刷新保持当前会话
- 支持对话收藏（个人专属）
- 对话按模块自动归类
- 模块化时间线展示

### 4.2 技术栈

**前端**
- Vue 3 + Composition API
- Vite 5（构建工具）
- Element Plus（UI 组件库）
- Vue Router 4（路由）
- Pinia（状态管理）
- Axios（HTTP 客户端）+ SSE（流式接收）

**后端**
- FastAPI（Web 框架）
- SQLAlchemy 2.0（ORM）
- SQLite（关系型数据库）
- Pydantic（数据校验）
- JWT（用户认证）
- python-docx / fpdf2（文档生成）

**AI 能力**
- DeepSeek API（大语言模型）
- OpenSearch（向量数据库）
- sentence-transformers（Embedding）
- LangGraph（图编排框架）
- 自研 Agent 工具集

### 4.3 RAG 完整链路

```
用户问题 → 查询重写 → 向量检索 → Rerank 精排
        ↓
   可选：联网搜索
        ↓
   构建上下文（带长度截断）
        ↓
   LLM 流式生成
        ↓
   来源溯源 + 答案返回
```

关键环节说明：

1. **查询重写**：多轮对话中补全省略/指代（如"它"→"报销制度"）
2. **混合检索**：向量检索 + 关键词检索（OpenSearch 支持）
3. **Rerank 精排**：用 LLM 对初检结果重新打分
4. **来源溯源**：每个回答标注 [1][2] 引用编号，关联原始文档

### 4.4 Agent 架构

```
用户输入 → 意图理解 → 工具选择 → 工具调用
        ↑                       ↓
        └────── 结果整合 ←───────┘
```

支持 Function Calling，工具集支持：
- `web_search` - 联网搜索
- `calculate` - 数学计算
- `create_document` - 生成 Word/PDF
- `get_current_date` - 日期时间
- `analyze_file` - 文件分析

并具备**幻觉兜底机制**：检测 LLM 是否"假装"调用了工具但实际没有，强制补救。

### 4.5 LangGraph 高级编排

- **状态管理**：跨轮次上下文传递
- **多步推理**：任务拆解 → 子任务执行 → 结果聚合
- **异常处理**：步骤失败自动重试或回退
- **人机协同**：关键节点可注入用户决策

## 五、亮点与差异化

### 🌟 亮点 1：模块化知识库 + 智能路由

四个业务模块 + 一个自由问答，**每个模块独立知识库**。用户选择模块后，系统自动从对应知识库检索，避免跨模块信息污染。自由问答模块支持用户自建个人知识库。

### 🌟 亮点 2：三模态推理引擎无缝切换

同一界面下，用户可一键切换 **RAG / Agent / LangGraph** 三种推理范式：
- RAG：日常知识问答
- Agent：需要联网/计算/工具的任务
- LangGraph：多步复杂任务

每种模式共享同一份知识库和上下文，无需重复登录/切换。

### 🌟 亮点 3：流式响应 + 工具调用可视化

采用 SSE 流式响应，AI 的每一步思考、生成、工具调用结果都**实时呈现**，告别"长时间黑屏"。附件（生成的 Word/PDF）即时可下载。

### 🌟 亮点 4：完整的 RAG 工程化链路

从文档上传、解析、清洗、语义切分、向量化、混合检索、Rerank、来源溯源，覆盖 RAG 全流程。不是简单调 API，而是**工程级**的 RAG 系统。

### 🌟 亮点 5：细粒度权限管理

- **管理员**可创建共享知识库（所有用户可见）
- **普通用户**仅能创建个人知识库（仅自己可见）
- **模块级**权限隔离（普通用户无法删除/编辑初始模块库）
- **JWT** 认证 + 基于角色的访问控制（RBAC）

## 七、快速开始

### 7.1 环境要求

- Python 3.11+
- Node.js 18+
- OpenSearch 2.x
- Redis 7.x
- DeepSeek API Key

### 7.2 后端启动

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate   # Windows
pip install -r requirements.txt

# 配置环境变量（.env）
DEEPSEEK_API_KEY=your_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
TAVILY_API_KEY=your_tavily_key

# 启动
python main.py
```

### 7.3 前端启动

```bash
cd frontend
npm install
npm run dev
```

访问 `http://localhost:5173` 即可使用。
